Frequently Asked Question

the difference between var, let, and const
var has function scope, is hoisted with undefined, and allows reassignment. let has block scope, is hoisted without initialization, and allows reassignment. const has block scope, is hoisted without initialization, and cannot be reassigned. Avoid var; prefer let and const.
the difference between map(), forEach(), and filter()
map() creates a new array by transforming each element. forEach() loops through elements but doesn’t return a new array. filter() creates a new array with elements that meet a condition.
explain arrow functions and how they are different from regular functions
Arrow functions are shorter syntax for functions. making them ideal for callbacks but unsuitable for methods needing this.
H how JavaScript Promises work
JavaScript Promises handle asynchronous tasks. They have three states: **pending**, **fulfilled**, or **rejected**. Use `.then()` for success, `.catch()` for errors, and `.finally()` for cleanup after completion.
how closures work in JavaScript
Closures occur when a function retains access to its lexical scope (variables) even after the outer function has finished executing, allowing the inner function to use those variables later..
